# OTP Space — X-проекты: Handoff для Claude Code

## Что это

Интерактивный HTML-прототип раздела «X-проекты» в корпоративной системе OTP Space.
Два файла, оба нужны в одной папке:

- `xproject.html` — страница X-проекта со списком целей
- `goal_detail.html` — страница отдельной цели с навигацией по дереву


### Главная колонка (main-col)
1. Заголовок проекта + кнопка «Развернуть»
2. Описание проекта
3. DOS проекта (ul)

## Компонент: таблица целей

### Структура данных одного квартала
```js
{
  id: 'q1',
  label: 'Q1 2026',
  status: 'Завершено',
  sc: '#EAF3DE',  // background статус-бейджа
  st: '#27500A',  // text цвет статус-бейджа
  rows: [ /* массив целей */ ]
}
```

### Структура строки цели
```js
{
  id: 'g1',
  level: 0,            // 0=корень, 1=дочерняя, 2=внучка
  type: 'ind',         // 'ind' | 'cmd'
  tags: ['x'],         // опц: дополнительные теги ['x' = Х-цель]
  name: 'Название цели',
  goalWeight: '20%',   // вес ЦЕЛИ в проекте (в колонке Вес)
  owners: ['Рощина Наталья | HR', 'Симоненко Сергей | IT', ...],
  weights: ['20%', '15%', ...], // вес УЧАСТИЯ каждого (weights[i] → owners[i])
  plan: '45 дней',
  fact: '46 дней',
  factClass: 'late',   // 'ok' | 'late' | 'empty'
  hasChildren: true,
  open: false,         // состояние раскрытия
  children: [ /* вложенные строки той же структуры */ ]
}
```

### Поведение строк
- **Шеврон `›` слева** — раскрывает/сворачивает дочерние строки (только expand, без навигации)
- **Стрелка `›` справа** — переход на `goal_detail.html?goalId=...&goal=...&proj=...`
- **Клик по строке** — тоже expand/collapse, без перехода
- Разделитель между строками верхнего уровня: тонкая строка `.tr-sep`
- «+ ещё N целей» если >5 строк (константа `SHOW_LIMIT=5`)

### Колонка «Вес»
- Показывается только для `level === 0`
- Берётся из `row.goalWeight`
- Отображается как фиолетовый бейдж `.weight-badge`
- Если `owners.length > 1` — рядом иконка ⓘ с тултипом: «Это вес цели в проекте. У каждого исполнителя свой индивидуальный вес участия — смотрите внутри цели»

### Строка владельцев
```
[аватар владельца] Рощина Наталья | HR 20% | [стек аватаров со-вл.] 12 со-вл.
```
- Владелец: фиолетовый аватар (#EEEDFE/#3C3489), имя фиолетовое, личный вес из `weights[0]`
- Разделитель: тонкая вертикальная линия 1px
- Стек со-владельцев: серые аватары (#F1EFE8/#444441), до 4 видимых + счётчик +N
- **Ховер на стек** → попап `.co-stack-wrap` с микро-карточками каждого со-владельца

### Попап со-владельцев
- Позиционирование: `position: fixed` (не обрезается overflow)
- Координаты: вычисляются через `getBoundingClientRect()` в `showCoPopup(id, evt)`
- Открывается вверх или вниз в зависимости от места на экране
- Не закрывается при переводе курсора на сам попап (`keepCoPopup()`)
- Закрывается через 150мс после ухода курсора
- Каждая карточка: аватар 28px + имя + отдел + вес участия в бейдже

## Файл goal_detail.html

### Лэйаут
```
browser-chrome
  browser-bar
  app
    sidebar
    main
      topnav
      nav-bar (квартальные табы Q1/Q2 + stepper согласования)
      tree-nav (хлебные крошки — навигация по дереву целей)
      content (список карточек целей)
      bottom-bar (кнопка «Отозвать с согласования»)
```

### Дерево целей (TREE)
Полное дерево хранится в константе `TREE` — объект с вложенными `children`.
Каждый узел:
```js
{
  id: 'g1',
  type: 'ind' | 'cmd' | 'x' | 'project',
  name: '...',
  weight: '20%',       // вес в родительской цели
  pct: 100,            // % выполнения
  owner: 'Рощина Н.',
  dept: 'HR',
  quarter: 'Q1 2026',
  weightNum: '20',     // для поля «Вес, %»
  goalType: 'Количественная',
  strategicPriority: '...',
  period: 'Q1, 2026',
  planVal: '45', planUnit: 'дн',
  factVal: '46', factUnit: 'дн',
  comment: '...',
  dos: ['...'],        // Definition of Success
  dod: ['...'],        // Definition of Done (массив строк, '' = пустая строка-разделитель)
  children: [ ... ]
}
```

### Навигация по дереву
- Состояние: `curId` — текущий узел
- `navigateTo(id)` — обновляет `curId`, сбрасывает `expandedIds`, перерисовывает
- `getAncestors(id)` — возвращает путь от корня до `curId`
- Хлебные крошки: кликабельны, первый элемент (проект) возвращает на `xproject.html`

### Три секции на странице
1. **Родительская цель** — карточка уровня выше (если есть). Если нет — плашка «это цель верхнего уровня»
2. **Текущая цель** — раскрыта по умолчанию, border 1.5px
3. **Дочерние цели** — карточки с кнопкой «Провалиться ↓»

### Карточка цели
- Заголовок: шеврон + dot статуса + тег типа + название + прогресс-бар + вес + кнопка «Провалиться ↓» + иконка копирования
- Клик по шеврону — разворачивает детали (DoS, DoD, тип, вес, приоритет, период, оценка план/факт/комментарий)
- Кнопка «Провалиться ↓» — `navigateTo(node.id)`, хлебные крошки обновляются
- Кнопка «Свернуть» — сворачивает тело карточки

### Получение целей из URL
```js
var urlParams = new URLSearchParams(window.location.search);
var startId = urlParams.get('goalId') || 'g1';
```
`xproject.html` передаёт `goalId`, `goal`, `proj` через query string.
